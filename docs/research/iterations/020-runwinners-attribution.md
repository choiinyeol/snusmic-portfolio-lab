# 020 Run-Winners Attribution

Date: 2026-05-25
Status: accepted as mechanism evidence

## Idea

Iteration 018 found a large improvement from a small construction change: keep the PIT Fresh540 Top5 signal shell, but stop trimming still-valid winners back to equal weight at every rebalance.

Iteration 019 showed that this survived concentration, calendar, and slippage stress.

This iteration asks why it worked.

Hypothesis:

- equal-weight rebalancing was selling some winners too early;
- run-winners keeps those compounding positions while still ejecting rank-invalid names;
- if the excess return is broad across several winners, the rule is a portfolio-construction edge;
- if the excess return comes from one oversized position, the next mutation needs a max-position cap.

## Point-in-time contract

No new strategy signal is introduced here. The analysis reads only existing simulation artifacts generated by the PIT-only rules:

- `data/sim/summary.csv`
- `data/sim/monthly_holdings.csv`

- `data/sim/trades.csv`
- `data/sim/position_episodes.csv`
- `data/sim/equity_daily.csv`

The inspected accounts were generated by rules that use decision-date rank, report age, moving-average stack, 52-week-high distance, rebalance date, and configured costs only.

Future returns are used only after simulation completion for attribution and retrospective explanation, not for a trading decision.

## Buy rule

The analyzed rule is unchanged from Iteration 018:

- quarterly rebalance
- PIT trend candidates
- max report age 540 days
- 20/50/200MA stack required
- within 20% of the 52-week high
- Top20 persistence band
- 60-day minimum holding period
- Top5 target basket
- buy underweight selected names with available cash

## Sell/rebalance rule

The analyzed run-winners rule is unchanged:

- sell holdings absent from the selected rebalance basket;
- keep still-valid holdings inside the persistence band;
- do not sell a retained winner solely to restore equal weight;
- do not use leverage.

The comparison control is the equal-weight Fresh540 Top5 account, which mechanically sells down retained names back toward equal weight.

## Retrospective

The run-winners edge is not just one lucky stock.

The largest deltas are distributed across RFHIC, 에이피알, Lumentum, 티엘비, Corning, Palantir, and JYP. The strategy also loses less on weak retained names because cash is directed toward underweight selected names instead of constantly restoring all holdings to the same weight.

The key behavior is simple:

- equal-weight control realizes profits too eagerly;
- run-winners lets the market decide which winners become large;
- the PIT signal still controls membership, so this is not buy-and-forget drift;
- the risk-adjusted `vol50` variant keeps nearly the same return while reducing MDD from 29.74% to 26.82%.

The risk is also clear. Mature average top1 rises from 25.43% to 29.01%, and peak single-name weight reaches roughly 49.5%. That is high enough to monitor, but not yet enough to reject the construction because:

- Top3 concentration already failed in Iteration 019;
- the run-winners Top5 edge is broad across several names;
- MDD did not worsen versus the equal-weight Fresh540 control;
- the Vol50 variant improves drawdown without changing the sell-down rule.

Accept this as mechanism evidence. Do not replace the current best.

## Next mutation

Test whether the concentration risk can be reduced without killing the run-winners edge.

Candidate next experiments:

1. Hard max-position cap at 40% and 35% on rebalance.
2. Soft sell-down cap only when a retained name exceeds 45%.
3. Vol50 plus hard/soft cap combinations.
4. No cap, but add reporting: max single-name weight, top3 weight, and current concentration in the web portfolio page.

Important warning: a cap that constantly trims winners may simply recreate the equal-weight bug. The next mutation should treat caps as risk limits, not as another hidden rebalancing-to-equal-weight mechanism.
