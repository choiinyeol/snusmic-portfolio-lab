# Plain-language portfolio and accounting pass

이번 변경의 기준은 화면을 멋있게 보이게 만드는 것이 아니라, 개인투자자가 첫 화면에서 실제로 판단할 수 있는 정보를 남기는 것이다. 그래서 `스냅샷`, `원장`, `체결 원장`처럼 내부 개발자나 회계 담당자에게 가까운 말을 줄이고, 사용자가 바로 이해하는 `메인화면`, `포트폴리오`, `매매내역`으로 정리했다. 랜딩 페이지의 큰 문장도 구호형 문구를 제거하고 제품명과 기능 설명만 남겼다.

디자인 레퍼런스는 복제 대상이 아니라 원칙을 확인하는 용도로만 사용했다. 21st.dev 쪽에서는 작은 버튼, pill, muted text, dense list의 리듬을 확인했고, transitions.dev에서는 과한 장식보다 짧은 상태 변화만 쓰는 방향을 확인했다. Montage 계열 디자인 시스템에서는 토큰과 용어의 일관성이 제품 신뢰도를 만든다는 점을 참고했다. Claude advisor는 카드 벽을 줄이고 metric row, table, paragraph, hairline divider 중심으로 가자는 의견을 줬고, Gemini advisor는 `스냅샷/원장`보다 `메인화면/포트폴리오/매매내역`이 일반 사용자에게 맞다는 점과 파생 데이터 표기를 분명히 하라는 점을 재확인했다.

MTT #22의 현금 문제는 회계 오류라기보다 웹 artifact 누락 버그였다. `summary.csv`에는 `smic_mtt_strategy_top22`가 실현손익 약 3.568억 원, 현금 약 2.947억 원, 보유 평가액 약 1.571억 원을 갖고 있었지만, `current-holdings.json`에는 열린 포지션이 빠져 있었다. `position_episodes.csv`에는 187790.KS 25,798주가 열려 있었고, 평균 원가 6,364.1345원, 최근 가격 6,090원, 미실현손익 약 -707만 원이었다. 따라서 현금은 `입금 누계 + 실현손익 - 현재 보유 원가`로 설명된다.

앞으로 포트폴리오 화면에서는 실현손익과 현금을 같은 숫자로 보지 않는다. 실현손익은 이미 닫힌 매매에서 확정된 손익이고, 현금은 그 확정 손익과 입금액에서 아직 들고 있는 주식의 매입 원가를 뺀 잔액이다. 이 관계를 `accounting-reconciliation.json`으로 저장해 화면과 테스트가 같은 검산식을 보도록 했다. 사용자가 이상하다고 느낀 지점은 숨기지 않고, 포트폴리오 화면 안에서 줄글과 수식형 라인으로 설명한다.

생성·파생 데이터는 결론처럼 포장하지 않는다. 이번 검산 데이터는 원천 거래와 열린 포지션에서 계산한 derived artifact이며, 파생이라는 사실을 UI 본문에 명시한다. 원천 데이터가 부족한 통계, 예를 들어 발간 후 n일 대기 매수 우위나 목표가 도달 후 추가 보유 성과는 아직 결론으로 쓰지 않고 `추가 artifact 필요`로 둔다.
