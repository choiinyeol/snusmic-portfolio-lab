# Page Patterns

## Dashboard Page

Use this structure:

1. Header with useful title, plain-language subtitle, status metadata, and primary action.
2. KPI summary strip with compact cards.
3. Primary chart or table.
4. Secondary diagnostic panels.
5. Recent activity or data quality panel.

Do not lead with a vague marketing hero on analytical pages.

## Financial Detail Page

Use this structure:

1. Entity name, ticker or ID, status badge, last updated timestamp.
2. Metric strip with 4 to 8 key metrics.
3. Main analytical area with chart first and explanation beside or below it.
4. Detail tabs or sections for holdings, signals, trades, logs, and diagnostics.
5. Footer metadata for data source, refresh time, and known limitations.

## Signal Review Page

Use this structure:

1. Plain-language goal: what report evidence is being reviewed and why.
2. Compact metric strip: target hit rate, current return, progress, and recent action.
3. Main table: the full report universe with sorting and compact default columns.
4. Current candidates: rows that still need human review.
5. Data quality and caveats below the main workflow.

Do not invent strategy names or buy/sell logic. If the data only supports "review candidate", say that.

Avoid separate controls that restate the same concept. If a board already has a full-table universe, do not add another top-level tab group for "all / active / hit" unless each mode changes the user's next action.

## Statistics Page

Use this structure:

1. Compact header with sample date, sample count, ticker count, and evaluation window.
2. Executive summary with 4 to 6 metrics and one sentence explaining the main read.
3. Distribution and concentration panels before long charts.
4. Whole-sample map with a short reading guide.
5. Representative price paths and diagnostic tables below.

Statistics pages should explain what the chart proves before asking the user to interpret every visual manually.

## Table-Heavy Page

Use sticky headers when tables are long. Right-align numeric cells. Use compact rows. Add short explanations for non-obvious factors. Provide meaningful empty states.
