#### 1. 투자의견 Buy와 목표주가 82,900원 – PER Valuation

PER Valuation 선택 근거 및 결과

쿠콘의 22F 적정 주가를 도출하기 위해 PER Valuation을 선정했다. 데이터 서비스 사업부 의 성장 및 페이먼트 사업부의 견고한 매출 발생에 따른 22F 순이익 증가를 전망하여 PER Valuation을 사용했다. 동사의 22F 예상 EPS를 4,889원으로 추정한다. 산출한 예상 EPS에 PER Multiple 16.96배를 곱해 적정주가 82,900원과 상승여력 55.90%를 도출했다.

Peer Group 선정 쿠콘의 적정 주가를 산출하기 위해 Intuit, 웹케시, 아이티아이즈, 세틀뱅크, NHN를 Peer 그룹으로 선정했다. Intuit은 데이터 분석 서비스와 페이먼트 서비스 사업을 영위하는 핀테 크 기업으로, 동사와 유사한 사업 모델을 보유하기 때문에 향후 주가추이 유사할 것으로 판 단하여 Peer로 선정했다. 웹케시는 동사의 모회사로 B2B 핀테크 사업을 영위하며 동사와 같이 마이데이터 산업 확장에 따른 수혜를 받을 것으로 판단해 Peer로 선정했다. 아이티아 이즈는 클라우드 서비스를 바탕으로 금융 분야로 사업을 확대하고 있어 동사와 유사한 주가 추이를 보일 것으로 판단해 Peer로 선정했다. 세틀뱅크는 동사의 페이먼트 사업부와 동일 한 간편결제 및 전자금융 플랫폼 서비스를 영위하여 Peer로 선정했다. NHN은 동사의 고객 사로, 간편결제 서비스 사업을 영위하며 주가 추이가 유사하여 Peer로 선정했다. NHH은 결 제 서비스 외에도 게임, 커머스 등 매출 구성이 다양하여, NHN 매출 내의 결제 서비스 매출 비중으로 디스카운트하여 12m Fwd P/E를 산출했다. Plaid는 API 사업을 영위하여 동사와 사업부문이 동일하지만 비상장사이기 때문에 기각했다. Yodlee는 동사와 동일한 데이터를 분석 사업을 영위하지만 비상장사이기 때문에 기각했다. NICE평가정보의 경우 수익 모델 이 유사하지만 주가추이가 상이하여 기각했다. 이에 따라 선정한 각 기업의 12m Fwd P/E 를 통해 Average P/E 16.96배를 산출했다.

#### RIM Valuation

#### 2. 산업분석

데이터 3법과 마이데이터 데이터 3법의 개정으로 마이데이터 사업이 가능해지면서 소비자는 간소화된 과정을 통해 각종 기관으로부터 개인정보를 전달받을 수 있다. 데이터 3법은 데이터 활성화를 촉진하는 개인정보보호법, 신용정보법, 정보통신망법을 통칭하는 용어로, 데이터 3법이 개정됨에 따 라 개인을 식별할 수 없도록 처리된 가명정보를 데이터 분석 및 관련 서비스 개발에 활용할 수 있게 되었다. 이는 개인정보를 모아 소비자에게 유용한 맞춤 서비스를 제공하는 마이데 이터 사업이 가능해짐을 의미하며, 마이데이터 사업자들은 마이데이터 사업자들은 소비자 에게 API를 통해 서비스를 제공해야 한다.

소비자는 마이데이터 앱 및 서비스를 통해 마이데이터 사업자에게 개인정보를 요청하여, 개 인정보를 보유하고 있는 여러 정보 제공자로부터 개인정보를 한 번에 전달받거나 타인에게 전송할 수 있다. 마이데이터 사업자를 통해 소비자가 개별 기관에 각각 접근해야했던 개인 정보 요청 과정이 간소화됨으로써 소비자는 보다 효율화된 과정을 거쳐 편리하게 정보를 전 달받거나 전송할 수 있다.

데이터 세상의 통로, API

API는 기업이나 기관 사이 데이터가 이동하는 통로의 역할을 하는 동시에 상품의 형태로 판 매되는 가공된 데이터를 의미한다. 22년 1월부터 API를 통한 정보 전달이 의무화되었으며, API 상품 분야가 다양화됨에 따라 상품 수도 증가할 전망이다.

소비자로부터 정보 전송을 요청받은 마이데이터 사업자는 은행, 공공기관, 기타기관 등의 정보 제공자에게 정보 전송 요구를 전달하고, 정보 제공자는 API 형태의 개인정보를 마이데 이터 사업자에게 전송한다. 소비자로부터 정보를 요구받아 정보제공자에게 요구를 전달하 는 마이데이터 사업자엔 KB국민은행, 신한은행, 뱅크샐러드 등이 있으며, 데이터의 이동 통 로인 API를 제공하는 API 서비스 업체에는 쿠콘이 있다. 쿠콘은 오랜 업력을 통한 국내 최

- 다 고객사 확보와 빠른 마이데이터 사업자 허가 획득으로 시장에서 독보적인 국내 1위를 선 점하고 있다.

API 방식은 개인이 마이데이터 앱이나 사이트에 직접 로그인해 필요한 정보를 다운받거나 제 3자에게 전송을 허용하면, 개인정보가 API를 통해 전달되는 방식이다. 해당 방식은 스크 래핑 방식의 허점을 보완할 수 있는데, 스크래핑 방식은 기업이 은행, 공공기관 등의 사이트 에 개인 대신 접속하여 화면에 있는 개인정보를 추출해오기 때문에 정보의 과잉 수집 가능 성이 제기된다는 문제점이 있다. 정부는 개인이 타인에게 정보 접근 권한을 양도하지 않고 직접 정보에 접근해 다운로드 및 전송할 수 있는 API 방식을 의무화함으로써 개인의 정보 보안 및 통제를 강화하고자 했다. 현재는 API 상품 분야가 금융 분야로 제한적이지만 데이

- 터 개방 및 활용을 위한 법적 기반이 마련됨에 따라 의료, 유통 등의 분야로 확대되어 여러 분야의 데이터가 결합된 상품이 생성될 전망이다.

API를 통한 마이데이터 서비스 제공의 의무화로 인해 증가한 API의 수요가 업계 1위인 쿠 콘으로 유입되어 동사의 고객사 수가 18년 1,260개에서 22F 2,200개로 증가할 것으로 예 상하며, 이에 따라 마이데이터향 매출액이 22F 185억 원(+84.60%, YoY)까지 성장할 것으 로 전망한다.

마이데이터 사업이 허가제로 시행되며 개인 정보를 다루는 마이데이터 관련 사업을 영위하 고자 하는 기업들은 마이데이터 사업자 본허가를 획득해야 한다. 이에 따라 마이데이터 사 업자 본허가를 획득한 기업 수가 1Q21 28개에서 3Q21 40개로 증가했다. 국내 API 제공 기 업 1위인 동사의 데이터 보유량은 50,000종으로 2위 기업인 기웅정보통신의 15,000종과 비교해 압도적으로 높은 보유량이다. 데이터 보유량은 제공 가능한 상품 수이자, 고객사의 필요를 충족시킬 가능성을 의미하므로 본허가를 획득한 신규 시장 진입자들의 API 수요가 시장의 독보적 1위인 동사로 유입될 것을 전망한다. 또한 본허가 획득을 대기 중인 예비허 가 획득 기업들이 추후 본허가를 획득하여 신규 시장 진입자가 되면 시장 내 독점적 지위를 지닌 동사의 API를 이용할 가능성이 높아 추가적인 수요 발생을 기대할 수 있다. 이에 따라 고객 수가 증가하며 동사의 매출액 증대를 전망한다.

쿠콘의 전자금융 API는 최다 금융기관의 전용망 확보를 바탕으로 한 신규 고객사 증가 추세 와 캡티브 고객사들이 유지되는 산업적인 특징으로 인해 22F 매출액이 180억원(+9.88%, YoY)까지 성장할 것으로 전망한다. 전자금융 API는 전방 산업 고객사들의 입금이나 출금 등의 금융거래를 돕는다. 금융기관의 전용망을 더 많이 확보할 수록 고객사들의 거래 편의 성이 높아지기 때문에 금융기관 전용망의 수가 곧 API 산업 내의 경쟁력이다. 동사는 은행 23개, 증권사 21개 등 경쟁사 세틀뱅크보다 많은 전용망을 확보하고 있다. 동사는 국내 최 다 전용망을 확보해 동사의 전자금융 API 이용 시 모든 금융사와 거래가 가능하다는 장점이 있다. 이러한 장점을 바탕으로 동사의 고객사는 19년 160개에서 21년 343개로 증가하고 있 는 추세이다.

한편, 전자금융 API는 서비스를 구축하기 위한 초기 투자비용이 크다. 기존에 이용하던 API 에서 다른 API로 교체하기 위해서는 약 1.2억 원 이상의 비용이 소모되고 서비스 구축을 위 한 시간도 약 8개월가량 소모된다. 즉 고객사들이 기존에 이용하던 업체에서 다른 업체의 API로 교체해야 할 유인이 없기 때문에 캡티브 고객사가 유지되는 특징이 있다. 이러한 산 업적 특징으로 인해서 캡티브 고객사향의 수수료 매출 역시 유지될 것으로 전망한다.
